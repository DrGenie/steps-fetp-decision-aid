1. Overview of the preference study and STEPS

The STEPS tool is built on a discrete choice experiment (DCE) that asked stakeholders to choose between alternative Field Epidemiology Training Program (FETP) designs and an explicit opt out option. Each training option was described by:

  • Programme tier: Frontline, Intermediate or Advanced.
  • Career incentive: government and partner certificate, university qualification, or government career pathway.
  • Mentorship intensity: low, medium or high.
  • Delivery mode: blended, fully in person, or fully online.
  • Expected response time for events: within 30, 15 or 7 days.
  • Cost per trainee per month in Indian rupees.

In each DCE task respondents viewed two FETP options (Training A and Training B) plus an opt out alternative where no new training option was funded. By observing how respondents traded off attributes and cost across many tasks, the study estimated preference parameters that describe how attractive each feature is on average and how this differs across people.

STEPS links three ingredients:

  1. Preference estimates from the DCE (a mixed logit model and a two class latent class model).
  2. Cost templates from WHO, NIE and NCDC for different FETP tiers, including opportunity cost of trainee time where available.
  3. Simple epidemiological multipliers that translate training outputs into indicative public health benefits.

These inputs are combined to produce, for any user selected configuration:

  • Predicted endorsement and opt out rates.
  • Willingness to pay for attribute improvements.
  • Programme cost per cohort and economic cost including opportunity cost.
  • Indicative monetary benefits, net benefit and benefit cost ratio.
  • National scale results when the configuration is rolled out to multiple cohorts.
STEPS is a decision support tool, not a full transmission model. It focuses on transparent, easily explained indicators that can be used in policy discussions and business case documents and it can be aligned with more detailed modelling where needed.

2. Preference model and utilities

The main model used in STEPS is a mixed logit (MXL) model estimated on the DCE data. The model includes:

  • An alternative specific constant for Training A (ASC_A).
  • An alternative specific constant for the opt out option (ASC_OptOut).
  • Dummy variables for each non cost attribute level relative to a baseline.
  • A linear cost term measured in thousands of rupees per trainee per month.

For individual n, alternative j and task t, the systematic utility in the mixed logit specification is:

U(n, j, t) = ASC terms + attribute coefficients × attribute levels + cost coefficient × cost

In the STEPS code the mixed logit mean coefficients (measured at the sample average) are:

2.1 Building utility from attribute levels

In the STEPS code, the systematic part of utility for a programme configuration is constructed in two steps:

  1. Compute a non cost component that adds up the constant and all non cost attribute coefficients.
  2. Add the cost term, which is the cost coefficient multiplied by cost per trainee per month in thousands.

The mixed logit non cost utility is:

nonCostUtility = ASC_A + tier coefficient + career incentive coefficient + mentorship coefficient + delivery mode coefficient + response time coefficient

The cost term is:

costUtility = cost coefficient × (cost per trainee per month in INR / 1,000)

Total utility for the FETP programme relative to Training B is:

V_FETP = nonCostUtility + costUtility
Example 1: Non cost utility for an Advanced, high mentorship programme

Consider an Advanced FETP configuration with:

  • Programme tier: Advanced.
  • Career incentive: university qualification.
  • Mentorship intensity: high.
  • Delivery mode: blended (reference).
  • Expected response time: within 7 days.

Using the mixed logit means, the non cost utility is:

nonCostUtility = ASC_A + Advanced + UniQual + MentorHigh + Blended (0) + Resp7 = 0.168 + 0.487 + 0.017 + 0.640 + 0 + 0.610 = 1.922

3. Endorsement probabilities

STEPS reports endorsement as the predicted share of stakeholders who would support the chosen programme configuration instead of choosing the opt out alternative. To compute this, STEPS compares the utility of the FETP configuration with the utility of opting out.

Let V_FETP denote the total utility of the configured option relative to Training B, and ASC_OptOut denote the opt out constant. The difference in utility between FETP and opt out is:

deltaV = -ASC_OptOut + nonCostUtility + costUtility

With a single FETP option and an opt out, the endorsement probability is:

P(endorse) = exp(deltaV) / (1 + exp(deltaV)) P(opt out) = 1 - P(endorse)
Example 2: Endorsement for the Advanced high mentorship configuration

Continue the configuration from Example 1 and assume a cost per trainee per month of 75,000 INR. The cost term is:

cost per trainee per month = 75,000 INR cost in thousands = 75,000 / 1,000 = 75 cost coefficient = -0.005 costUtility = -0.005 × 75 = -0.375

Total utility for FETP is:

V_FETP = nonCostUtility + costUtility = 1.922 - 0.375 = 1.547

The opt out constant is ASC_OptOut = -0.601, so the utility difference between FETP and opt out is:

deltaV = -ASC_OptOut + V_FETP = -(-0.601) + 1.547 = 0.601 + 1.547 = 2.148

The endorsement probability is:

P(endorse) = exp(2.148) / (1 + exp(2.148)) ≈ 8.57 / 9.57 ≈ 0.895 or 89.5 percent

STEPS would display this as roughly 90 percent endorsement and 10 percent opt out. If the cost slider is moved upward, the costUtility term becomes more negative and endorsement falls. If cost is reduced, endorsement increases.

STEPS applies exactly the same logic when the supportive latent class group is selected, but substitutes the Class 2 parameters in place of the mixed logit means. This allows users to explore endorsement among a more supportive audience.

4. Willingness to pay (WTP)

Willingness to pay expresses each attribute coefficient in rupee terms per trainee per month. For a DCE with a linear cost term, the marginal willingness to pay for attribute level k is:

WTP_k = - beta_k / beta_cost

Here beta_k is the coefficient on the attribute level and beta_cost is the cost coefficient measured per thousand INR. To convert into rupees per trainee per month, we multiply by 1,000.

4.1 WTP from the mixed logit model

With a cost coefficient of -0.005, the mixed logit implies the approximate WTP values in the table below.

Attribute level Coefficient WTP (thousand INR) WTP (INR per trainee per month)
Intermediate programme 0.220 44.0 44,000
Advanced programme 0.487 97.4 97,400
Medium mentorship 0.453 90.6 90,600
High mentorship 0.640 128.0 128,000
Fully in person (relative to blended) -0.232 -46.4 -46,400
Fully online (relative to blended) -1.073 -214.6 -214,600
Response within 15 days 0.546 109.2 109,200
Response within 7 days 0.610 122.0 122,000
Example 3: WTP for high mentorship

The coefficient for high mentorship is 0.640. With a cost coefficient of -0.005 the WTP is:

WTP_highMentor = - 0.640 / (-0.005) = 128 (thousand INR) = 128,000 INR per trainee per month

This means that, averaged across respondents, high mentorship is valued as much as an increase of about 128,000 INR per trainee per month in programme cost. If actual cost is lower than this, high mentorship tends to improve cost effectiveness as long as it also improves epidemiological outcomes.

4.2 WTP from the supportive latent class group

For Class 2 of the latent class model the cost coefficient is -0.001. WTP values are therefore larger for a given attribute coefficient. For example, the WTP for an Advanced programme among supportive stakeholders is roughly:

WTP_Advanced_Class2 = - 0.422 / (-0.001) ≈ 422 (thousand INR) ≈ 422,000 INR per trainee per month

STEPS uses these WTP values to generate the WTP summary shown in the Results tab. They complement endorsement by indicating the monetary value stakeholders attach to different design features.

WTP is based on stated preferences in a survey context and should not be interpreted as a literal budget willingness to pay by any single organisation. It is best used as a relative measure when comparing design options and prioritising attributes.

5. Programme and economic costs

STEPS builds programme and economic costs at two levels:

  • Per cohort, based on cost per trainee per month chosen in the Configuration tab and programme duration.
  • Across cohorts, based on the number of cohorts entered by the user.

5.1 Programme cost per cohort

For a given configuration, programme cost per cohort (excluding opportunity cost) is:

C_prog = costPerTraineePerMonth × traineesPerCohort × durationMonths
Example 4: Programme cost per cohort

Suppose the configuration is:

  • Programme tier: Advanced (24 months).
  • Cohort size: 20 trainees.
  • Cost per trainee per month: 250,000 INR.

Then:

C_prog = 250,000 × 20 × 24 = 250,000 × 480 = 120,000,000 INR

5.2 Economic cost and opportunity cost

Cost templates from WHO, NIE and NCDC identify:

  • Shares of cost allocated to staff, travel, materials, supervision and overheads.
  • Where applicable, an opportunity cost component for trainee time.

When the opportunity cost toggle is off, STEPS reports:

  • Programme cost per cohort based on the slider and cohort parameters.
  • Allocation of this cost across components using the chosen template shares.

When the opportunity cost toggle is on, STEPS:

  • Adds the template specific opportunity cost per cohort to programme cost.
  • Reports total economic cost as the sum of programme and opportunity cost.

Economic cost per cohort is:

C_economic = C_prog + C_opp
Example 5: Economic cost with opportunity cost

Take the Advanced programme example with C_prog = 120,000,000 INR. Assume the cost template indicates that opportunity cost of trainee time is equivalent to 20 percent of programme cost:

C_opp = 0.20 × 120,000,000 = 24,000,000 INR C_economic = 120,000,000 + 24,000,000 = 144,000,000 INR

STEPS will show 144,000,000 INR as total economic cost per cohort when opportunity cost is included.

5.3 Cost components

Component specific amounts are obtained by multiplying template shares by the programme cost per cohort. For example, if staff are 40 percent of programme cost, then:

staffCost = 0.40 × C_prog

Similar calculations are carried out for travel, materials, supervision and overhead categories. These are reported in the Costing details tab and updated each time the user changes cost per trainee, cohort size, programme tier or template source.

6. Epidemiological multipliers and benefits

STEPS uses simple multipliers to translate FETP training into public health outcomes. These multipliers are specific to each tier:

  • Graduates per cohort as a share of trainees.
  • Outbreak responses per cohort per year.
  • Indicative monetary value per graduate (INR).
  • Indicative monetary value per outbreak response (INR).

Default values are conservative and can be changed in the Advanced settings tab or in the epi_config.json file that STEPS can read at start up.

6.1 Graduates and outbreak responses

Let:

  • gradRate_tier be the graduation rate (share) for the tier.
  • outbreaks_tier be outbreak responses per cohort per year.
  • endorse be the endorsement probability for the configuration.

For one cohort the expected number of graduates is:

graduates_per_cohort = traineesPerCohort × gradRate_tier × endorse

Expected outbreak responses supported per year are:

outbreaks_per_year_per_cohort = outbreaks_tier × endorse

6.2 Monetary benefits

Let:

  • valueGrad_tier be the indicative value per graduate (INR).
  • valueOutbreak_tier be the indicative value per outbreak response (INR).

Then benefits per cohort are:

benefit_graduates = graduates_per_cohort × valueGrad_tier benefit_outbreaks = outbreaks_per_year_per_cohort × valueOutbreak_tier × planningHorizonYears totalBenefit_per_cohort = benefit_graduates + benefit_outbreaks
Example 6: Benefits for an Advanced programme

Assume:

  • Advanced tier graduation rate: 0.95.
  • Outbreak responses per cohort per year: 0.8.
  • Planning horizon: 5 years.
  • Value per graduate: 600,000 INR.
  • Value per outbreak response: 40,000,000 INR.
  • Cohort size: 20 trainees.
  • Endorsement: 0.90 (90 percent).

Then expected graduates per cohort are:

graduates_per_cohort = 20 × 0.95 × 0.90 = 20 × 0.855 = 17.1 graduates

Graduate related benefits are:

benefit_graduates = 17.1 × 600,000 = 10,260,000 INR

Outbreak responses per year are:

outbreaks_per_year_per_cohort = 0.8 × 0.90 = 0.72

Over the five year planning horizon:

benefit_outbreaks = 0.72 × 40,000,000 × 5 = 0.72 × 200,000,000 = 144,000,000 INR

Total indicative benefit per cohort is:

totalBenefit_per_cohort = 10,260,000 + 144,000,000 = 154,260,000 INR

7. National simulation

The National simulation tab scales up cohort level results to the national level using the number of cohorts specified in the Configuration tab. The scaling is direct and transparent.

7.1 Costs at national level

Let:

  • C_economic be economic cost per cohort.
  • benefit_per_cohort be total indicative benefit per cohort.
  • N_cohorts be the number of cohorts.

Then:

totalCost_national = C_economic × N_cohorts totalBenefit_national = benefit_per_cohort × N_cohorts netBenefit_national = totalBenefit_national - totalCost_national

The national benefit cost ratio is:

BCR_national = totalBenefit_national / totalCost_national

This ratio is the same as the per cohort ratio because the scaling factor cancels out.

7.2 Epidemiological metrics at national level

National level graduates and outbreak responses are:

graduates_national = graduates_per_cohort × N_cohorts outbreaks_national = outbreaks_per_year_per_cohort × N_cohorts

These are reported in the National simulation tab and update whenever the cohort size, number of cohorts, endorsement or epidemiological multipliers are changed.

Example 7: National scaling of the Advanced scenario

Using the Advanced example above:

  • Economic cost per cohort: 144,000,000 INR.
  • Benefit per cohort: 154,260,000 INR.
  • Graduates per cohort: 17.1.
  • Outbreak responses per year per cohort: 0.72.

Suppose the configuration is scaled to 10 cohorts. Then:

totalCost_national = 144,000,000 × 10 = 1,440,000,000 INR totalBenefit_national = 154,260,000 × 10 = 1,542,600,000 INR netBenefit_national = 1,542,600,000 - 1,440,000,000 = 102,600,000 INR

The benefit cost ratio is:

BCR_national = 1,542,600,000 / 1,440,000,000 ≈ 1.07

National graduates and outbreak responses are:

graduates_national = 17.1 × 10 = 171 graduates outbreaks_national = 0.72 × 10 = 7.2 outbreak responses per year

8. Worked example matching the user interface

This section walks through a complete example that matches a realistic configuration in the STEPS interface. The goal is to show, in one place, how the preference model, costs and epidemiological multipliers interact.

8.1 Configuration

Consider the following configuration, which matches a typical scenario used in workshops:

  • Programme tier: Advanced (24 months).
  • Career incentive: university qualification.
  • Mentorship intensity: high.
  • Delivery mode: blended.
  • Expected response time: within 7 days.
  • Cost per trainee per month: 75,000 INR.
  • Cohort size: 20 trainees.
  • Number of cohorts: 10.
  • Preference model: mixed logit average.
  • Opportunity cost: included.

8.2 Step 1: Utility and endorsement

From Example 1 the non cost utility is 1.922. With cost per trainee per month of 75,000 INR:

costUtility = -0.005 × (75,000 / 1,000) = -0.005 × 75 = -0.375 V_FETP = 1.922 - 0.375 = 1.547

The opt out constant is -0.601, so:

deltaV = -ASC_OptOut + V_FETP = 0.601 + 1.547 = 2.148 P(endorse) ≈ 0.895 (89.5 percent) P(opt out) ≈ 0.105 (10.5 percent)

STEPS will therefore show endorsement close to 90 percent for this configuration.

8.3 Step 2: Programme and economic costs

Programme cost per cohort is:

C_prog = 75,000 × 20 × 24 = 75,000 × 480 = 36,000,000 INR

Suppose the chosen cost template implies an opportunity cost equal to 25 percent of programme cost. Then:

C_opp = 0.25 × 36,000,000 = 9,000,000 INR C_economic = 36,000,000 + 9,000,000 = 45,000,000 INR per cohort

8.4 Step 3: Epidemiological multipliers and benefits

Assume:

  • Advanced graduation rate: 0.95.
  • Outbreak responses per cohort per year: 0.8.
  • Value per graduate: 600,000 INR.
  • Value per outbreak response: 40,000,000 INR.
  • Planning horizon: 5 years.

With endorsement of 0.895 and cohort size of 20:

graduates_per_cohort = 20 × 0.95 × 0.895 ≈ 17.03 graduates

Graduate benefits are:

benefit_graduates = 17.03 × 600,000 ≈ 10,218,000 INR

Outbreak responses per year are:

outbreaks_per_year_per_cohort = 0.8 × 0.895 ≈ 0.716

Over five years:

benefit_outbreaks ≈ 0.716 × 40,000,000 × 5 ≈ 0.716 × 200,000,000 ≈ 143,200,000 INR

Total indicative benefit per cohort is:

totalBenefit_per_cohort ≈ 10,218,000 + 143,200,000 ≈ 153,418,000 INR

8.5 Step 4: Benefit cost ratio and net benefit

Per cohort:

netBenefit_per_cohort ≈ 153,418,000 - 45,000,000 ≈ 108,418,000 INR BCR_per_cohort ≈ 153,418,000 / 45,000,000 ≈ 3.41

8.6 Step 5: National scaling

With 10 cohorts:

totalCost_national = 45,000,000 × 10 = 450,000,000 INR totalBenefit_national ≈ 153,418,000 × 10 ≈ 1,534,180,000 INR netBenefit_national ≈ 1,534,180,000 - 450,000,000 ≈ 1,084,180,000 INR

Graduates and outbreak responses are:

graduates_national ≈ 17.03 × 10 ≈ 170 graduates outbreaks_national ≈ 0.716 × 10 ≈ 7.2 responses per year

These values are close to what the STEPS National simulation tab will show for the same configuration.

9. Advanced settings and sensitivity

The Advanced settings tab exposes key epidemiological and display parameters so that technical users can align STEPS with national assumptions without editing code. The main adjustable parameters are:

  • Graduation rates for each tier.
  • Outbreak responses per cohort per year for each tier.
  • Value per graduate for each tier.
  • Value per outbreak response for each tier.
  • Planning horizon in years.
  • Display exchange rate (INR per USD).

Changing any of these parameters immediately updates:

  • Per cohort epidemiological metrics and monetary benefits.
  • National simulation results.
  • Benefit cost ratio and net benefit in the Results and National simulation tabs.

For stability, the tool does not change the underlying preference coefficients when advanced settings are edited. Only the translation between graduates, outbreak responses and monetary benefits is affected.

Example 8: Increasing the value per outbreak response

Suppose a new analysis suggests that each outbreak response avoided has a value of 50,000,000 INR instead of 40,000,000 INR. If the user updates the value per outbreak response for the relevant tier:

  • Graduate benefits remain unchanged.
  • Outbreak related benefits increase by 25 percent.
  • Total benefits and net benefits increase.
  • The benefit cost ratio increases proportionally to the change in benefits.

This allows country teams to align STEPS with updated numbers from health security or burden of disease models while keeping the preference based endorsement structure intact.

The epi_config.json file provides the same parameters in a structured format that can be edited before deployment. When present, the tool reads this file on load and uses its contents as defaults. If the file is not present, built in default values are used.

10. Recommended use and limits

STEPS is designed as a transparent decision aid for use with ministries, partners and funders such as the World Bank, WHO and CDC. It is best used to:

  • Compare alternative FETP programme designs and identify high value options.
  • Support business case documents with credible endorsement, cost and benefit metrics.
  • Structure workshop discussions around trade offs between cost, endorsement and epidemiological impact.

It is not intended to replace:

  • Detailed epidemiological models of disease transmission.
  • Full economic evaluations that include discounting of costs and health outcomes over long horizons.
  • Comprehensive fiscal planning tools for multi programme portfolios.
In practice, STEPS works best when used alongside detailed modelling. STEPS can highlight promising configurations quickly. Detailed models can then be applied to a shortlist of options if required for large scale investment decisions.
